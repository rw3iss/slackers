# docs/

Project documentation, organised by purpose.

| Folder | What lives here |
|---|---|
| [architecture/](architecture/) | Current-state architecture notes — package layout, TUI model split, P2P stack, data flow. Update when the shape of the codebase changes. |
| [features/](features/) | Feature deep-dives — friends, notifications, themes, secure mode, search, etc. One doc per subsystem. |
| [development/](development/) | Build, test, and contribution workflow — how to add a new overlay, how to run two P2P instances locally, how to add a theme. |
| [archive/](archive/) | Landed plans and completed audit reports. Historical context only — read to understand *why* the code looks how it does, not as a to-do list. |
| [superpowers/](superpowers/) | Internal agent-driven workflow plans (not user-facing). |

For user docs, see `README.md` and `How_It_Works.md` at the repo root.
For Claude-specific working context, see `CLAUDE.md` at the repo root.
